# 24. 面试与练习题

## 学习目标

- 回顾 SwiftUI 基础与进阶核心知识点
- 通过题目巩固“会写”和“会解释”两种能力
- 建立后续持续练习路径

## 中文讲解

学习到这一步，建议用“题目驱动复盘”来巩固能力。面试和实战都不只是考 API 记忆，更看重你是否理解状态驱动、数据流、组件化和可维护性。

练习时可以采用三步法：先口述思路，再手写最小实现，最后说明为什么这样设计。这个过程会显著提升你的表达和实战稳定性。

如果你准备面试，建议把每个主题都准备一个“可运行小例子 + 2 分钟讲解版本”。

## English Notes

At this stage, question-driven review is a strong way to consolidate your learning. Interviews and real work both care more about reasoning than raw API memorization.

Use a three-step exercise loop: explain your approach, implement a minimal solution, and justify design choices.

For interviews, prepare one runnable mini-example and a two-minute explanation for each topic.

## 示例题目

```text
1. Explain the difference between @State and @Binding with a code example.
2. Build a list-detail flow using NavigationStack.
3. Add persistence for a simple user preference.
4. Refactor duplicated UI into a reusable component.
5. Describe one common SwiftUI performance pitfall and how to avoid it.
```

## 常见错误

- 只会写代码，不会解释设计取舍
- 只刷题，不做完整小项目复盘
- 对状态流转描述含糊

## 本章小结

你学会了如何用练习题系统复盘 SwiftUI 能力，并为面试或实战输出做准备。

What you learned: you now have a structured way to review and present your SwiftUI skills.
