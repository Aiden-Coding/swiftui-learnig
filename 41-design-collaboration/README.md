# 41. 与设计协作

## 学习目标

- 理解设计协作中的关键交付物
- 学会把设计稿转成可实现需求
- 能减少开发和设计之间的返工

## 中文讲解

高质量 UI 不是开发单方面完成的，设计协作需要统一组件、状态、边距和交互规则。明确规格会大幅降低返工。

入门建议固定同步节奏：开发前对齐、开发中核对、提测前走查。

## English Notes

Great UI quality depends on design-engineering collaboration. Shared component and interaction rules reduce rework.

Adopt a simple cadence: align before build, verify during build, review before QA.

## 示例代码

```text
Design Handoff Checklist
1. Screen states defined
2. Spacing and typography tokens defined
3. Interaction and transitions defined
```

## 常见错误

- 只对齐静态稿，不对齐状态稿
- 交互规则口头约定无记录
- 提测前没有联合走查

## 本章小结

你学会了开发与设计协作的最小流程。

What you learned: you can now run a practical design handoff workflow.
