# 06. 列表与数据展示：List、ForEach

## 学习目标

- 学会把数组数据显示成列表。
- 理解 `List` 和 `ForEach` 的配合方式。
- 能搭出一个最基础的课程列表页面。

## 场景引入

真实 App 里经常要显示课程列表、消息列表、待办事项、设置项。这时你不能手动一条条写死，而要学会“根据数据重复生成界面”。

## 本章术语先看懂

- `List`：列表容器
- `ForEach`：遍历数据并生成视图
- `id`：每一项的唯一标识

## 一句话理解

`List` 负责像列表一样显示，`ForEach` 负责把数据一条条变成界面。

## 手把手操作步骤

1. 创建 `CourseListView`。
2. 粘贴代码并运行。
3. 观察 4 行课程是否都被正确渲染。
4. 再自己增加两条课程数据。

## 完整示例代码

```swift
import SwiftUI

struct CourseListView: View {
    let courses = ["Swift Basics", "Layout", "State", "Navigation"]

    var body: some View {
        List {
            ForEach(courses, id: \.self) { course in
                HStack {
                    Image(systemName: "book.fill")
                        .foregroundStyle(.blue)
                    Text(course)
                }
            }
        }
    }
}

#Preview {
    CourseListView()
}
```

## 代码拆解（小白重点）

- `courses` 是数据来源。
- `List` 提供滚动列表容器。
- `ForEach` 把数组里的每项变成一行界面。
- `id: \.self` 适合字符串数组。

## 新手排错流程

1. 列表为空时，先检查数组里有没有内容。
2. `ForEach` 报错时，检查 `id` 是否合适。
3. 每行样式混乱时，考虑把单行内容抽成独立视图。

## 章节小测（带答案）

### 题 1

`List` 和 `ForEach` 分别负责什么？

参考答案：`List` 负责列表容器，`ForEach` 负责遍历数据并生成视图。

### 题 2

为什么这里可以用 `id: \.self`？

参考答案：因为数组元素是字符串，可以直接把字符串本身作为唯一标识。

### 题 3

为什么每一行用了 `HStack`？

参考答案：因为图标和文字是左右排列的。
