# 45. 数据分析基础：先知道用户在做什么，才能知道该优化什么

## 学习目标

- 理解为什么产品优化离不开基础数据分析。
- 知道哪些行为值得记录，哪些指标值得先看。
- 建立“先观察，再优化”的基础意识。

## 场景引入

很多开发者在做优化时，容易只凭感觉。比如：

- 我觉得这个按钮应该很常用
- 我觉得这个页面问题不大
- 我觉得这个功能用户应该喜欢

但真实情况常常和感觉不一样。数据分析的价值，就是帮你更客观地看到：

- 用户真的点了什么
- 哪一步最容易流失
- 哪个功能使用频率最高
- 哪个新功能几乎没人用

## 本章术语先看懂

- `埋点`：记录用户在某个关键节点上的行为。
- `指标`：用于衡量产品表现的数据，比如点击率、完成率。
- `漏斗`：一个流程中各步骤逐渐流失的情况。
- `行为数据`：用户实际操作留下的记录。

## 一句话理解

优化的前提不是“我觉得”，而是“我知道用户在做什么”。

## 最适合先观察的内容

1. 核心功能的点击和使用次数。
2. 用户完成一个关键流程时，在哪一步退出最多。
3. 新功能上线后，实际使用的人多不多。
4. 某个页面打开很多，但下一步转化很低。

## 新手常见误区

- 什么都想埋，最后数据一团乱。
- 只记数量，不看流程和上下文。
- 有数据但没有明确问题意识，不知道该怎么用。

## 新手排错流程

1. 先定义最关键的 1-2 条用户路径。
2. 只围绕这些路径记录最有价值的事件。
3. 看到异常数据时，先结合页面体验分析，不要只盯数字。

## 章节小测（带答案）

### 题 1

为什么产品优化不能只靠感觉？

参考答案：因为感觉容易偏差，真实用户行为往往和主观判断不同。

### 题 2

最值得先记录的埋点通常是什么？

参考答案：与核心功能和关键流程相关的关键节点行为。

### 题 3

为什么“什么都埋”反而可能没价值？

参考答案：因为噪音太多会让真正重要的信息更难识别和分析。

## 练习任务

- 基础练习：为你的项目选出 3 个最关键的用户行为事件。
- 加强练习：画出一个简单的关键流程漏斗。
- 挑战练习：假设某个按钮点击率低，列出 3 个可能原因和验证方式。

## 本章小结

学完这章后，你应该已经知道：数据分析的重点不是数字越多越好，而是先看清用户真正做了什么，再决定怎么优化。
