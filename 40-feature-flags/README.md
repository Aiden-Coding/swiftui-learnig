# 40. 功能开关基础：不是所有新功能都要一次性全部放开

## 学习目标

- 理解功能开关为什么在真实项目里很常见。
- 知道功能开关能帮助团队控制风险。
- 建立对“灰度、试验、逐步放量”思路的基础认知。

## 场景引入

如果你做了一个新功能，直接一次性对所有用户开放，一旦出问题，影响面会很大。功能开关的价值，就是让你可以更灵活地控制：

- 先只给内部测试
- 先给一小部分用户
- 发现问题后快速关闭
- 准备好后再全面开放

## 本章术语先看懂

- `功能开关`：控制某个功能是否启用的机制。
- `灰度发布`：先让一部分用户使用新功能。
- `风险控制`：在问题发生时降低影响范围。
- `放量`：逐步扩大可用范围。

## 一句话理解

功能开关的意义，不是让功能更复杂，而是让发布更稳、更可控。

## 一个简单例子

假设你做了一个“AI 学习建议”功能：

- 你不一定要第一天就对所有用户打开
- 可以先只给测试用户使用
- 如果建议结果不稳定，可以先关闭
- 调整好后再继续放量

## 功能开关最适合解决什么问题

- 新功能还不够稳定
- 希望逐步发布而不是一次性全开
- 不同用户可能看到不同能力
- 需要快速回滚某个功能表现

## 新手常见误区

- 觉得功能开关只适合超大型团队。
- 所有功能都想加开关，结果系统越来越乱。
- 有开关但没有明确默认策略和关闭策略。

## 新手排错流程

1. 先判断这个功能是否真的存在发布风险或试验需求。
2. 只给高风险或需要灰度的功能加开关。
3. 设计好“关闭后用户看到什么”的兜底体验。

## 章节小测（带答案）

### 题 1

功能开关最核心的价值是什么？

参考答案：帮助团队控制新功能发布风险，让功能可以逐步开放或快速关闭。

### 题 2

为什么不是所有功能都值得加开关？

参考答案：因为过多开关会增加复杂度和维护成本，只有高风险或需要灰度的功能更适合。

### 题 3

功能开关和一次性全量发布相比，最大优势是什么？

参考答案：能更灵活地试验、观察和回滚，降低出问题时的影响范围。

## 练习任务

- 基础练习：列出你项目中 2 个适合加功能开关的能力。
- 加强练习：为其中一个功能设计“关闭时的兜底体验”。
- 挑战练习：思考功能开关和 A/B 实验的区别与联系。

## 本章小结

学完这章后，你应该已经知道：功能开关不是为了让系统更复杂，而是为了让新功能上线更稳、更可控。
