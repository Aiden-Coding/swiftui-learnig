# 30. CI/CD 基础：让构建和检查更自动化

## 学习目标

- 理解 CI/CD 在开发流程中的作用。
- 知道持续集成和持续交付分别解决什么问题。
- 对自动构建、自动检查建立基础认知。

## 场景引入

如果每次改完代码都要手动做这些事：

- 自己记得跑测试
- 自己手动检查格式
- 自己手动打包
- 自己提醒别人可以测了

那流程就很容易出错，也很依赖个人记忆。CI/CD 的目标，就是把这些重复、容易漏掉的步骤尽量自动化。

## 本章术语先看懂

- `CI`：持续集成，常见是自动构建、自动测试、自动检查。
- `CD`：持续交付或持续部署，常见是自动发布或自动分发。
- `流水线`：把多个自动化步骤串成一条流程。
- `自动化检查`：例如代码格式检查、单元测试、构建验证。

## 一句话理解

CI/CD 不是替代开发，而是帮团队把重复流程自动化、标准化。

## 一个典型的 CI 流程会做什么

1. 代码提交到仓库。
2. 自动触发构建。
3. 自动运行测试。
4. 自动检查是否有明显问题。
5. 结果反馈给开发者。

## 一个典型的 CD 流程会做什么

1. 在通过测试和构建后生成可分发版本。
2. 自动上传到测试渠道或内部环境。
3. 某些团队还会继续自动发布到正式环境。

## 为什么新手也值得理解 CI/CD

即使你现在是一个人做项目，也会慢慢遇到这些问题：

- 忘记跑测试
- 提交了不能构建的代码
- 改完以后不知道当前版本是否稳定

CI/CD 的思路，会帮助你更早建立“流程化交付”的习惯，而不是只会本地跑通。

## 新手常见误区

- 以为 CI/CD 只和大团队有关。
- 觉得自动化只是锦上添花，不影响质量。
- 还没把基础检查做好，就急着上复杂发布流水线。

## 新手排错流程

1. 先明确你最想自动化的第一件事是什么，比如自动构建或自动测试。
2. 不要一口气把所有流程全自动化，先从最基础的检查开始。
3. 流水线报错时，先看它失败在构建、测试还是配置阶段。

## 章节小测（带答案）

### 题 1

CI 最常见解决的是什么问题？

参考答案：自动构建、自动测试和自动检查，减少低级错误流入主分支。

### 题 2

CD 通常和什么更相关？

参考答案：和版本分发、交付、部署更相关。

### 题 3

为什么新手也值得理解 CI/CD？

参考答案：因为它能帮助建立更稳定的交付习惯，而不是只关注本地能不能跑。

## 练习任务

- 基础练习：列出你项目里最适合自动化的 3 个步骤。
- 加强练习：为自己的小项目写一份“提交前检查清单”。
- 挑战练习：尝试了解一个常见 CI 平台会如何执行构建和测试。

## 本章小结

学完这章后，你应该已经知道：CI/CD 的核心不是工具名，而是把重复、关键的质量流程自动化，让交付更稳定。
