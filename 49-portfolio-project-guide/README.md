# 49. 作品集项目指南：做项目不是堆功能，而是展示能力

## 学习目标

- 理解作品集项目为什么和练习项目不完全一样。
- 知道一个好的作品集项目最该展示哪些能力。
- 建立“少而完整”优于“多而零散”的项目选择意识。

## 场景引入

很多新手在准备作品集时，容易走两个极端：

- 做了一堆小页面，但看不出完整能力
- 想做一个特别大的项目，结果最后做不完

作品集项目的重点，不是证明你功能做得多，而是让别人快速看出你会什么、能做到什么程度。

## 本章术语先看懂

- `作品集项目`：用于展示你能力和思考方式的代表性项目。
- `完整度`：项目从功能、结构到体验是否相对完整。
- `展示价值`：别人是否能快速看出项目亮点和你的能力。
- `讲述能力`：你是否能把项目目标、方案和取舍讲清楚。

## 一句话理解

好的作品集项目不是功能最多，而是最能清楚展示你的能力边界。

## 一个作品集项目最值得展示什么

1. 你能不能把一个需求做完整。
2. 你是否考虑了状态、错误、边界和体验。
3. 你是否有基本的数据流和结构组织能力。
4. 你能不能解释自己为什么这样设计。

## 更推荐怎样选题

- 选一个你能做完整的小到中型项目。
- 题目最好有明确用户场景，不只是组件拼盘。
- 让项目能覆盖你想展示的几个关键能力。

例如：

- 学习打卡应用
- 课程管理应用
- 笔记与任务结合的效率工具

## 新手常见误区

- 觉得项目越大越厉害。
- 做了很多页面，但没有一个真正完整闭环。
- 只会展示界面，不会解释思路和取舍。

## 新手排错流程

1. 选题时先问：这个项目能完整做完吗？
2. 功能规划时先保核心闭环，再加加分项。
3. 项目完成后，整理一版自己的讲述提纲，而不是只给别人看代码。

## 章节小测（带答案）

### 题 1

为什么作品集项目不一定越大越好？

参考答案：因为太大容易做不完，反而不如一个完整度高的小中型项目更能展示能力。

### 题 2

作品集项目最核心要展示什么？

参考答案：展示你解决问题、组织代码、处理边界和讲清思路的能力。

### 题 3

为什么只展示很多零散页面不够？

参考答案：因为零散页面很难体现你是否真的能完成一个完整场景和闭环功能。

## 练习任务

- 基础练习：为自己列出 2 个适合作品集的项目题目。
- 加强练习：为其中一个项目设计“核心闭环功能”。
- 挑战练习：写一段 2 分钟项目介绍，练习怎么讲清目标、亮点和取舍。

## 本章小结

学完这章后，你应该已经知道：作品集项目最重要的不是炫，而是能让别人清楚看到你真正具备哪些能力。
