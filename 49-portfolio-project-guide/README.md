## 学习前准备

- 打开 Xcode，准备一个可运行的 SwiftUI 工程。
- 保证你能看到实时预览或模拟器运行结果。
- 建议边看边敲，不要只读不练。

## 核心概念（小白版）

输出能体现能力的完整项目。

如果你是新手，可以先记住一个原则：**先把最小功能跑通，再逐步加复杂度**。不要一开始就追求“写得很高级”，先把“写得正确、看得懂”放在第一位。

### 这一章你真正要会的 3 件事

1. 能说清这个知识点在真实项目里什么时候会用到。
2. 能写出最小示例，并知道每一行在做什么。
3. 出错时能按步骤排查，而不是盲改代码。

## 手把手操作步骤

1. 新建一个练习视图，例如 Chapter49PracticeView。
2. 先把本章示例代码完整粘贴并运行，确认可见结果。
3. 改一个小点（文案、颜色、布局、状态），确认你真的能控制它。
4. 再加一个小交互（按钮点击、开关切换、输入更新等）。
5. 如果报错，先看报错行，再回看本章“常见问题”。
6. 运行通过后，完成本章“练习任务”。

## 完整示例代码

```swift
import SwiftUI

struct Chapter49PracticeView: View {
    @State private var counter = 0

    var body: some View {
        VStack(spacing: 16) {
            Text("第 49 章：作品集项目指南")
                .font(.headline)

            Text("当前计数：\(counter)")
                .foregroundStyle(.secondary)

            Button("点我 +1") {
                counter += 1
            }
            .buttonStyle(.borderedProminent)
        }
        .padding()
    }
}
```
## 代码拆解（逐段理解）

- @State private var counter = 0：定义一个会变化的本地状态。
- VStack(spacing: 16)：把内容纵向排列，并设置间距。
- Text(...)：显示章节信息和当前状态。
- Button(...)：触发交互，点击后修改状态。
- 当 counter 变化时，界面会自动刷新显示新值。

- 基础练习：补全项目 README 的架构与取舍说明。
- 加强练习：把按钮样式、文案、布局改成你自己的版本。
- 挑战练习：新增一个状态变量，让界面出现第二种反馈。

## 常见问题（新手高频）

### 1) 预览没反应怎么办？

先确认代码能编译通过，再尝试重新运行预览；如果仍不更新，重启预览或改用模拟器运行。

### 2) 为什么我改了代码但界面不变？

优先检查你改的是不是当前显示的视图；其次检查是否有状态驱动更新。

### 3) 什么时候算学会这一章？

当你能不看文档写出最小示例，并且能做一个小扩展，就算真正入门。
