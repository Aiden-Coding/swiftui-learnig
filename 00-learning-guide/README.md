# 00. 学习指南：怎么把这套 SwiftUI 教程学成真正的能力

## 这份指南是给谁准备的

如果你打开教程时，脑子里常常会冒出这些问题：

- 我应该按什么顺序学？
- 每章都要完整敲代码吗？
- 学到后面开始变难，是不是我不适合？
- 为什么我“看懂了”，但一写还是不会？

那这份学习指南就是专门给你的。

它的目的不是讲某个 API，而是帮你把这套教程用得更有效。

## 先记住一件很重要的事

学 SwiftUI 最常见的误区，不是学得慢，而是学得太散。

很多人会这样学：

1. 今天看一个按钮教程
2. 明天看一个导航教程
3. 后天看一个动画教程
4. 一周后发现每个都“好像看过”，但还是拼不成完整页面

这不是你不行，而是学习路径断了。

所以这套教程才会刻意按顺序拆成：

- 基础页面能力
- 真实项目能力
- 工程化基础
- 产品与协作
- 长期成长路线

## 最推荐的学习节奏

### 第一步：按顺序，不要一开始跳太多

如果你是新手，最推荐的是至少先完整学完 `01-10`。

因为这 10 章解决的是所有后续章节的地基问题：

- 页面怎么搭
- 修饰符怎么理解
- 状态为什么会驱动界面
- 列表、导航、表单怎么组成真实页面

如果这里不稳，后面你会不断“见过，但不会用”。

### 第二步：每章至少做 3 件事

不要只看完就关掉，尽量每章都做这 3 件事：

1. 把示例代码亲手敲一遍
2. 改一两个变量或文案观察变化
3. 用自己的话复述“这一章到底在解决什么问题”

这一步特别关键。因为“看懂”和“会用”之间，差的往往就是这点亲手实践。

### 第三步：每完成 5 章就做一次小复盘

你可以问自己：

- 我现在已经能独立做什么页面？
- 哪一章我看得懂，但一写就卡？
- 哪个知识点最容易混？
- 如果现在让我做一个小练习，我最想从哪一章回去翻？

这能帮你防止学着学着又散掉。

## 学习时最容易误判的几个点

### 误判 1：我看懂了，所以我会了

很多人看教程时会有一种错觉：页面代码看着顺，解释也看懂了，于是以为自己掌握了。

但真正检验会不会的方法只有一个：

- 关掉教程
- 自己重新写一次

如果你能在不照抄的前提下写出大致结构，那才是真正开始变成你的能力。

### 误判 2：学到后面变难，就是我基础差

不一定。

很多时候不是你退步了，而是教程内容开始进入更接近真实开发的阶段了。

比如你从：

- 文本和按钮
- 进入到状态管理、异步加载、共享数据、架构

这本来就会更难。

难是正常的，不代表你不适合，只代表你进入下一层了。

### 误判 3：我应该先把所有概念学完再做项目

这通常会拖得很久，而且效果不好。

更推荐的方式是：

- 先学一段
- 然后立刻做一点
- 再带着问题回来看教程

项目不是“最后才做”，项目本身就是学习的一部分。

## 什么时候该开始做项目

我的建议很明确：

- 学完 `01-10` 后，就可以开始做很小的页面组合练习
- 学完 `11-20` 后，就应该开始做一个小而完整的项目

不要等到“我全会了再开始做”。

因为很多能力，只有在项目里才会暴露出来：

- 状态到底怎么组织
- 页面为什么越来越乱
- 错误和边界该怎么处理
- 哪些组件值得复用

## 一套很适合新手的学习方式

你可以直接照着这条路线走：

### 第 1 段

完成 `01-05`

目标：

- 理解 SwiftUI 是什么
- 理解 View、修饰符、布局和状态

### 第 2 段

完成 `06-10`

目标：

- 能做列表、导航、表单和模态交互

### 第 3 段

完成 `11-18`

目标：

- 把异步、复用、生命周期和小项目串起来

### 第 4 段

完成 `19-30`

目标：

- 开始建立工程化意识

### 第 5 段

完成 `31-50`

目标：

- 建立产品、协作和长期成长视角

## 遇到卡住时怎么办

如果你现在卡住了，不要马上怀疑自己，先按下面顺序排查：

1. 我是没看懂概念，还是不会写出来？
2. 我是卡在语法，还是卡在页面结构？
3. 我是不是跳章节太快了？
4. 我是不是只看没写？
5. 我能不能先退回前一章或相关章节再看一次？

很多学习卡点，不是因为内容太难，而是因为前置环节没走扎实。

## 这一套教程最推荐的使用姿势

你可以把它当成：

- 一套系统课程
- 一份练习清单
- 一个项目中遇到问题时的回查资料库

最不推荐的使用方式是：

- 只收藏
- 只看标题
- 一口气翻很多章但从不练

## 最后的提醒

学 SwiftUI 这件事，不是谁一下子把所有 API 背下来，谁就赢。

真正会拉开差距的是：

- 你有没有持续动手
- 你有没有做过完整项目
- 你有没有复盘自己反复出错的地方

所以别急，按顺序走，扎实一点，比学得很快但很散更重要。
