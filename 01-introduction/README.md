# 01. SwiftUI 是什么与开发环境准备

## 学习目标

- 知道 SwiftUI 是用来做什么的。
- 理解“声明式界面”到底是什么意思。
- 能完成 Xcode 基础环境准备，并跑通第一个 SwiftUI 页面。
- 建立一个很重要的学习心态：先把地基打稳，再往后学控件和项目。

## 为什么这一章一定要先学

很多新手一上来就急着写按钮、写列表、写动画，但还没搞清楚 SwiftUI 到底是什么。结果就是：

- 代码能抄下来，但不知道为什么这样写
- 一个报错就卡住，因为不知道问题出在哪一层
- 页面能显示，但脑子里没有整体结构

这一章的作用，就是先把地基打稳。你不用一下子学很多 API，只需要先明白三件事：

1. SwiftUI 是 Apple 提供的现代界面框架。
2. 你写的是“界面应该长什么样”，而不是“界面一步步怎么做出来”。
3. 学 SwiftUI 前，先把开发环境跑通，比记住很多控件名字更重要。

## 本章术语先看懂

- `SwiftUI`：Apple 的声明式 UI 框架，用来开发 iPhone、iPad、Mac 等平台界面。
- `View`：界面中的一个显示单元。它可以是一段文字、一个按钮、一张图片，甚至整个页面。
- `声明式 UI`：你直接描述“现在界面应该长什么样”，系统自己决定怎么渲染和更新。
- `Preview`：Xcode 中实时预览 SwiftUI 页面效果的功能。
- `修饰符`：写在 View 后面的配置方法，用来改变样式和行为。

## 一句话理解

SwiftUI 更像是在写“界面描述书”，你告诉系统页面应该是什么样子，系统去负责渲染和刷新。

## 先建立正确的学习心智模型

你可以把 SwiftUI 想成这样：

- 你不是在“手工拼装一个界面”
- 你是在“声明一个界面状态下应该看到什么”

这和很多刚接触编程的人直觉不太一样。因为我们平时更容易想到的是：

1. 创建一个标签
2. 改它的文字
3. 再改颜色
4. 再把它放到屏幕上

但 SwiftUI 更像这样：

```swift
Text("你好，SwiftUI")
    .font(.title)
    .foregroundStyle(.blue)
```

这段不是在命令系统“一步步创建一个 UILabel”，而是在描述：

- 这里有一段文字
- 它是标题样式
- 它是蓝色的

## SwiftUI 和传统写法有什么区别

如果你之前接触过 UIKit，可能会习惯这种思路：

- 先创建控件
- 再设置属性
- 再设置位置
- 最后加到页面里

而 SwiftUI 的核心区别是：

- 你更关注“页面现在应该显示什么”
- 而不是“页面内部一步步怎么搭起来”

这也是为什么后面你学到状态管理时，会感觉 SwiftUI 特别强调“状态变了，界面就变”。

## 学习前准备

开始这一章前，请确认你已经准备好：

1. 一台能安装 Xcode 的 Mac。
2. Xcode 15 或更新版本。
3. 能正常打开模拟器或者 Preview。
4. 知道怎么新建一个 SwiftUI App 工程。

如果你还不会创建工程，按下面步骤做：

1. 打开 Xcode。
2. 选择 `Create a new Xcode project`。
3. 选择 `App` 模板。
4. Interface 选 `SwiftUI`。
5. Language 选 `Swift`。
6. 创建项目后点击运行按钮。

### 预期结果

你应该能看到一个默认页面成功在模拟器或预览中显示出来。

## 手把手操作步骤

### 第 1 步：新建一个练习视图

在你的 SwiftUI 工程里，创建一个新的 Swift 文件，命名为 `WelcomeStarterView.swift`。

### 第 2 步：粘贴下面代码

## 完整示例代码

```swift
import SwiftUI

struct WelcomeStarterView: View {
    var body: some View {
        VStack(spacing: 12) {
            Text("你好，SwiftUI")
                .font(.title2)
                .fontWeight(.bold)

            Text("第一个页面运行成功")
                .foregroundStyle(.secondary)
        }
        .padding()
    }
}

#Preview {
    WelcomeStarterView()
}
```

### 第 3 步：运行预览或模拟器

如果你使用 Preview，应该马上看到两行文字：

- 第一行更大、更粗
- 第二行颜色更浅
- 两行上下排列，中间有一点间距

### 第 4 步：自己改一处

请你试着把：

- `"你好，SwiftUI"` 改成你自己的欢迎语
- 或者把 `.title2` 改成 `.largeTitle`
- 或者给页面再加一行自己的学习目标

### 预期结果

你修改的内容应该立刻体现在界面上。只要你能改动并看到结果，就说明你已经开始建立“代码和界面对应”的感觉了。

## 代码拆解（小白重点）

### `struct WelcomeStarterView: View`

这表示你定义了一个新的界面视图，名字叫 `WelcomeStarterView`。

你可以先把它理解成：

- `struct`：Swift 里的结构体
- `View`：SwiftUI 里的页面协议

也就是说：这个结构体是一个可以显示在屏幕上的界面。

### `var body: some View`

这行是 SwiftUI 最核心的入口。

你可以把它理解成：

- `body` 里面写什么
- 页面最终就会显示什么

注意这里不是函数，不用写括号，它是一个“计算属性”。

### `VStack(spacing: 12)`

`VStack` 表示把内容竖着排列。

这里面放了两个 `Text`，所以最后你会看到两行上下排列的文字。

`spacing: 12` 表示这两行之间间距是 12。

### 第一个 `Text`

```swift
Text("你好，SwiftUI")
    .font(.title2)
    .fontWeight(.bold)
```

这段表示：

- 显示一段文字
- 字号较大
- 字体加粗

### 第二个 `Text`

```swift
Text("第一个页面运行成功")
    .foregroundStyle(.secondary)
```

这里用了 `.foregroundStyle(.secondary)`，表示使用系统次级文字颜色。效果通常更灰一些，适合做说明文字。

### `.padding()`

给整个 `VStack` 增加内边距。没有它的话，文字会更贴近屏幕边缘，视觉上会更挤。

## 初学者最容易卡住的点

### 卡点 1：明明只有几行代码，为什么概念这么多

这是正常的。因为你现在学的不是“功能点”，而是在搭建之后所有章节都会反复用到的基础概念。

### 卡点 2：我看得懂代码，但自己写不出来

也正常。第一章最重要的目标不是“自己独立设计页面”，而是建立“看懂 + 改得动”的感觉。

### 卡点 3：Preview 和模拟器到底先用哪个

如果你只是改文字、颜色、布局，Preview 很方便。
如果你要验证交互和流程，模拟器更接近真实使用。

## 你现在真正学会了什么

学完这一章，你至少应该能回答：

1. SwiftUI 是什么？
2. `View` 是什么？
3. `body` 是干什么的？

## 新手排错流程（建议照着做）

### 情况 1：预览不显示

先检查：

1. 文件里有没有语法错误。
2. `#Preview` 是否写对。
3. Xcode 是否还在编译中。

### 情况 2：提示 `Cannot find type 'View' in scope`

通常是因为你漏写了：

```swift
import SwiftUI
```

### 情况 3：界面没变化

检查你改的是不是当前正在预览的那个 View，而不是改到了别的文件。

### 情况 4：代码没报错，但我还是不理解

这时候不要急着继续往后翻。建议你先做一件小事：

- 改 3 次文字
- 改 2 次字号
- 增 1 行内容

只要你能把这件事做顺，理解会明显更稳。
