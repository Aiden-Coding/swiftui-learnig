# 34. Widget 基础：把关键信息展示到桌面之外

## 学习目标

- 理解 Widget 和主应用页面的区别。
- 知道 Widget 更适合展示什么样的信息。
- 对 Widget 的使用场景建立基础认知。

## 场景引入

Widget 的价值，不是再做一个“小号 App 页面”，而是把最重要、最值得被快速看到的信息放到系统外层，比如主屏、锁屏等位置。

比如：

- 今日学习进度
- 今日待办数量
- 最近一条打卡记录
- 即将开始的课程提醒

这类内容都适合 Widget，因为用户不需要先打开 App 才能看到。

## 本章术语先看懂

- `Widget`：显示在系统界面上的轻量信息模块。
- `时间线`：Widget 在不同时间点展示内容的计划。
- `轻量展示`：信息要少而精，不能把整个页面塞进去。
- `入口价值`：用户一眼看见就能获得价值，或快速跳回 App。

## 一句话理解

Widget 不是“缩小版 App”，而是“最重要信息的快捷展示层”。

## 一个适合 Widget 的例子

如果你做的是学习打卡应用，Widget 可以显示：

- 今天已经完成了几项任务
- 还剩几项未完成
- 当前连续打卡天数

而不适合直接放进去的通常是：

- 很长的课程详情
- 复杂表单
- 多层交互流程

## 设计 Widget 时先想什么

1. 用户最想一眼看到什么？
2. 这条信息是否足够简短？
3. 它能否帮助用户快速决定下一步？
4. 它是否值得占用系统主屏位置？

## 新手常见误区

- 把 Widget 当成小页面复制过去。
- 信息太多，重点反而不清楚。
- 只考虑展示，不考虑用户看到后能获得什么价值。

## 新手排错流程

1. 如果 Widget 内容太杂，先删掉次要信息。
2. 如果用户一眼看不懂重点，重新确认核心指标是什么。
3. 如果 Widget 和 App 首页完全重复，想一想它是否真的提供了额外便利。

## 章节小测（带答案）

### 题 1

为什么 Widget 不适合放太复杂的内容？

参考答案：因为它的核心价值是快速展示关键信息，而不是承载完整交互流程。

### 题 2

什么类型的信息更适合放进 Widget？

参考答案：简短、核心、用户需要频繁查看的信息，比如进度、数量、提醒。

### 题 3

设计 Widget 时最该优先考虑什么？

参考答案：用户一眼能获得什么价值，以及最重要的信息是什么。

## 练习任务

- 基础练习：为你的学习应用设计一个 Widget 草图。
- 加强练习：只保留 3 条以内的核心信息重新思考展示内容。
- 挑战练习：把“今日进度 + 连续打卡天数”设计成一个更清晰的 Widget 信息结构。

## 本章小结

学完这章后，你应该已经知道：Widget 的重点不是功能多，而是能在最短时间里把最值得看的信息送到用户眼前。
