# 29. 组件库基础：把常用 UI 沉淀成团队资产

## 学习目标

- 理解组件复用和组件库的区别。
- 知道什么时候一个组件值得沉淀成“公共组件”。
- 能看懂一个基础的公共按钮组件示例。

## 场景引入

前面我们学过“抽组件”，那为什么还会有“组件库”这个概念？

区别在于：

- 抽组件更像解决当前页面的重复问题
- 组件库更像把常用、稳定的 UI 规范沉淀下来，供多个页面甚至多个项目复用

比如：统一主按钮、统一卡片、统一空状态页，这些都很适合做成公共组件。

## 本章术语先看懂

- `组件库`：集中管理可复用 UI 组件的集合。
- `公共组件`：多个页面都能共用的标准化界面单元。
- `封装`：把结构、样式和少量行为包装起来复用。

## 一句话理解

组件库不是把所有组件都公共化，而是把“稳定、常用、重复出现”的 UI 沉淀下来。

## 示例代码

```swift
import SwiftUI

struct PrimaryActionButton: View {
    let title: String
    let action: () -> Void

    var body: some View {
        Button(action: action) {
            Text(title)
                .fontWeight(.semibold)
                .frame(maxWidth: .infinity)
                .padding(.vertical, 12)
        }
        .buttonStyle(.plain)
        .foregroundStyle(.white)
        .background(Color.blue)
        .clipShape(RoundedRectangle(cornerRadius: 14))
    }
}

struct ComponentLibraryDemoView: View {
    var body: some View {
        VStack(spacing: 12) {
            PrimaryActionButton(title: "保存设置") {
                print("保存设置")
            }

            PrimaryActionButton(title: "开始学习") {
                print("开始学习")
            }
        }
        .padding()
    }
}

#Preview {
    ComponentLibraryDemoView()
}
```

## 代码拆解（小白重点）

- `PrimaryActionButton` 把主按钮的结构和样式统一封装了起来。
- 页面以后只需要传标题和点击行为，不用每次都重新写样式。
- 这类组件非常适合在项目中反复使用。

## 什么组件值得进入组件库

- 多个页面反复出现。
- 样式稳定，规则比较明确。
- 团队希望统一体验。
- 改一次希望全局都跟着生效。

## 新手常见误区

- 组件刚写出来就急着做成库，结果还没稳定。
- 把非常业务化的页面组件也强行公共化。
- 公共组件参数越来越多，反而更难用。

## 新手排错流程

1. 先判断这个组件是不是真的会在多个地方重复使用。
2. 参数太多时，先思考是不是职责已经超出“公共组件”范围。
3. 样式不统一时，检查是不是大家又各自写了一份按钮。

## 章节小测（带答案）

### 题 1

组件复用和组件库最大的区别是什么？

参考答案：组件复用更偏当前页面内的抽取，组件库更偏长期沉淀和跨页面统一复用。

### 题 2

什么样的组件更适合做成公共组件？

参考答案：稳定、常用、重复出现、需要统一风格的组件。

### 题 3

为什么不是所有组件都应该进组件库？

参考答案：因为很多组件只适合某个特定业务场景，强行公共化会增加复杂度。

## 练习任务

- 基础练习：给公共按钮增加禁用态样式。
- 加强练习：增加一个次级按钮组件。
- 挑战练习：为你的教程项目整理 3 个最值得沉淀的公共组件。

## 本章小结

学完这章后，你应该已经知道：组件库的价值，不在于组件数量多，而在于它能真正提高复用效率和界面一致性。

## 本章完成标准

学完这一章后，你至少应该能做到：

- 能分清“当前页面里的复用组件”和“值得沉淀成公共组件”的区别
- 能判断一个组件是否足够稳定、常用、值得进入组件库
- 能做一个最基础的公共按钮或卡片组件

## 学完这一章后马上可以做的练习

把你的小项目里最常用的一类控件提成公共组件，比如：

- 主按钮
- 标题卡片
- 空状态块

只要你真正整理出一类公共组件，就会更能理解“组件库”不是概念，而是长期资产。
