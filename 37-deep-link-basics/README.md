# 37. Deep Link 基础：让外部链接能直接打开指定页面

## 学习目标

- 理解 Deep Link 是什么，以及它为什么重要。
- 知道它和普通首页打开的区别。
- 建立对“外部入口直达页面”的基础认知。

## 场景引入

如果用户在消息、邮件、网页里点开一个链接，结果只能进入 App 首页，再自己慢慢找内容，体验通常不会太好。

Deep Link 的价值就在这里：让用户通过外部入口，直接到达对应内容，比如：

- 直接打开某门课程详情
- 直接进入某个活动页面
- 直接跳到某个任务详情

## 本章术语先看懂

- `Deep Link`：从外部入口直接跳到应用内部指定页面的能力。
- `路由`：根据链接决定要打开哪个页面。
- `外部入口`：比如网页、短信、邮件、二维码。
- `页面上下文`：打开后要展示的具体内容信息。

## 一句话理解

Deep Link 的核心价值，是减少用户从外部入口进入目标内容的路径成本。

## 一个简单理解例子

假设你的学习应用支持下面这种链接：

- `myapp://course/123`

用户点击后，应用不该只打开首页，而应该更理想地：

- 直接打开课程 `123` 的详情页

这就是 Deep Link 思维。

## 设计 Deep Link 时先想什么

1. 哪些页面最值得被外部直达？
2. 链接里最少需要哪些参数？
3. 如果目标内容不存在，要怎么兜底？
4. 如果用户还没登录，要先去哪里？

## 新手常见误区

- 只想“怎么跳过去”，不想“跳过去后缺什么上下文”。
- 目标页面不存在时没有兜底处理。
- 所有页面都想做 Deep Link，结果维护困难。

## 新手排错流程

1. 先从最核心的 1-2 个页面设计 Deep Link。
2. 确认链接携带的信息是否足够定位内容。
3. 设计好找不到内容、未登录等异常路径。

## 章节小测（带答案）

### 题 1

Deep Link 最核心的价值是什么？

参考答案：让用户从外部入口直接到达应用内的目标内容。

### 题 2

为什么目标内容不存在时也要设计兜底？

参考答案：因为真实环境里链接可能过期、参数错误或内容已删除。

### 题 3

为什么不建议所有页面都做 Deep Link？

参考答案：因为很多页面不值得外部直达，过多入口会增加复杂度和维护成本。

## 练习任务

- 基础练习：列出你应用里最适合做 Deep Link 的 3 个页面。
- 加强练习：为其中一个页面写出需要的链接参数。
- 挑战练习：设计“内容不存在时”的兜底跳转策略。

## 本章小结

学完这章后，你应该已经知道：Deep Link 的重点不是跳转本身，而是让外部入口和应用内部内容更顺畅地连接起来。
