# SwiftUI 从入门到项目实战

一套面向中文初学者的 SwiftUI 学习教程，目标不是只让你“看过概念”，而是尽量帮你走到这一步：

- 能看懂 SwiftUI 基础代码
- 能自己写出常见页面
- 能处理常见报错和边界情况
- 能把零散知识点串成完整项目

这套教程覆盖从界面基础、状态管理、异步加载，到工程化、产品化、团队协作与后续成长路线，适合想系统学习 SwiftUI 的同学按顺序长期推进。

## 这套教程适合谁

- 完全没有 SwiftUI 经验，想从零开始系统入门的人
- 学过一点 SwiftUI，但知识点零散、写项目容易卡住的人
- 想做一个更完整的 SwiftUI 学习路径，而不是只看零散文章的人
- 想把基础、项目、工程化意识一起补起来的人

## 学完后你会获得什么

如果你认真按顺序完成这套内容，你通常会逐步建立这些能力：

- 理解 SwiftUI 的基本写法和声明式思维
- 会用布局、状态、列表、导航、表单完成常见页面
- 会处理加载中、失败、空状态、持久化、共享数据等真实问题
- 对测试、性能、架构、发布、协作等主题建立基本工程意识
- 能开始做自己的 SwiftUI 小项目和作品集项目

## 推荐学习方式

这套教程最适合按“学一章，练一章，改一章”的节奏推进。

建议你这样学：

1. 先按顺序阅读，不要一开始跳太多章节。
2. 每章都亲手敲一遍代码，不只看不写。
3. 每完成 5 章，回头总结一次：我现在已经能独立做什么。
4. 遇到报错先看章节里的“排错流程”，再动手查问题。
5. 不要急着同时开很多项目，先做完一个小而完整的练习闭环。

## 每章你会看到什么

为了尽量对小白更友好，这套教程的大多数章节都会包含这些内容：

- 场景引入：先告诉你为什么要学这个
- 术语解释：先把关键名词讲清楚
- 一句话理解：帮助你快速抓住核心
- 完整示例：不是只有零散片段
- 代码拆解：告诉你每块代码在做什么
- 常见误区：提前帮你绕坑
- 排错流程：报错时先从哪里查
- 小测和练习：帮助你把“看懂”变成“会用”

## 建议从哪里开始

如果你是完全新手，直接从 [01-introduction/README.md](/D:/Documents/GitHub/swiftui-learnig/01-introduction/README.md) 开始按顺序学就可以。

如果你已经有一点 SwiftUI 基础，可以按你的目标切入：

- 想补基础页面能力：从 `01-10` 开始
- 想补真实项目能力：从 `11-20` 开始
- 想补工程化意识：从 `21-30` 开始
- 想补产品与协作能力：从 `31-50` 开始

## 这套教程更适合怎么使用

这不是一本适合“快速翻完”的资料，而更适合作为一个长期练习仓库来用。

你可以把它当成：

- 一套系统课程目录
- 一份持续练习清单
- 一个做 SwiftUI 小项目时随时回查的参考仓库

## 下一步建议

如果你已经学到一半以上，最值得做的事情通常不是继续收藏更多教程，而是开始做自己的项目，并把你已经学过的内容真正用进去。

推荐顺序：

1. 做一个完整小项目
2. 补上错误处理、空状态、持久化和基础测试
3. 再做一次重构，让结构更清楚
4. 最后整理成可以讲出来的作品集项目
